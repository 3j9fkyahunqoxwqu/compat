Abstract

This standard describes a collection of non-standard (and often vendor-prefixed) CSS properties and DOM APIs that web browsers need to support for compatibility with the de facto web.

1. Introduction

This section is non-normative.

There exists an increasingly large corpus of web content that depends on web browsers supporting a number of webkit-prefixed CSS properties and DOM APIs for functionality or layout. This holds especially true for mobile-optimized web content.

This specification aims to describe the minimal set of webkit-prefixed CSS properties and DOM APIs that user agents are required to support for web compatibility.

The Non-Standard Compatibility wiki page will serve as a rough guide for what this specification intends to cover.

2. Conformance

All diagrams, examples, and notes in this specification are non-normative, as are all sections explicitly marked non-normative. Everything else in this specification is normative.

The keywords "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "NOT RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in RFC 2119. For readability, these words do not appear in all uppercase letters in this specification. [RFC2119]

Requirements phrased in the imperative as part of algorithms (such as "strip any leading space characters" or "return false and terminate these steps") are to be interpreted with the meaning of the keyword ("must", "should", "may", etc.) used in introducing the algorithm.

Conformance requirements phrased as algorithms or specific steps may be implemented in any manner, so long as the end result is equivalent. (In particular, the algorithms defined in this specification are intended to be easy to follow, and not intended to be performant.)

3. DOM Interfaces

3.1. The WebKitCSSMatrix interface

[Constructor,
 Constructor(DOMString transformList),
 Constructor(WebKitCSSMatrix other)]
interface WebKitCSSMatrix : DOMMatrix {
    // Mutable transform methods
    WebKitCSSMatrix setMatrixValue(DOMString transformList);

    // Immutable transform methods
    WebKitCSSMatrix multiply(WebKitCSSMatrix other);
    WebKitCSSMatrix inverse();
    WebKitCSSMatrix translate(unrestricted double tx,
                              unrestricted double ty,
                              unrestricted double tz);
    WebKitCSSMatrix scale(optional unrestricted double scaleX,
                          optional unrestricted double scaleY,
                          optional unrestricted double scaleZ);
    WebKitCSSMatrix rotate(optional unrestricted double rotX,
                           optional unrestricted double rotY,
                           optional unrestricted double rotZ);
    WebKitCSSMatrix rotateAxisAngle(unrestricted double x,
                                    unrestricted double y,
                                    unrestricted double z,
                                    unrestricted double angle);
    WebKitCSSMatrix skewX(unrestricted double sx);
    WebKitCSSMatrix skewY(unrestricted double sy);
};

The WebKitCSSMatrix interface represents a 4Ã—4 homogeneous matrix with methods useful for 2D and 3D transform operations.

The WebKitCSSMatrix interface may be implemented as a subclass of the DOMMatrix and DOMMatrixReadOnly interfaces, defined in Geometry Interfaces Module Level 1 [geometry-1], with key differences noted below.

3.1.1. Constructors

matrix = new WebKitCSSMatrix()
Returns a new WebKitCSSMatrix object.

The WebKitCSSMatrix() constructor, when invoked, must run these steps:

  1. Let m be a new WebKitCSSMatrix object.
  2. For m, set the m11 element, m22 element, m33 element, m44 element to 1. Set the m12 element, m13 element, m14 element, m21 element, m23 element, m24 element, m31 element, m32 element, m34 element, m41 element, m42 element, m43 element to 0.
  3. Return m.
matrix = new WebKitCSSMatrix(transformList)
Returns a new WebKitCSSMatrix object.

The WebKitCSSMatrix(transformList) constructor, when invoked, must run these steps:

  1. Let m be a new WebKitCSSMatrix object.
  2. Return the result of the algorithm for the static DOMMatrixReadOnly fromString(transformList) method, as m.
matrix = new WebKitCSSMatrix(other)
Returns a new WebKitCSSMatrix object.

The WebKitCSSMatrix(other) constructor, when invoked, must run these steps:

  1. Let m be a new WebKitCSSMatrix object.
  2. For m, set the 16 elements m11 element â€¦ m44 element to the value of the 16 elements m11 element â€¦ m44 element of other.
  3. Return m.

WebKitCSSMatrix objects must inherit the following attributes from DOMMatrixReadOnly:

3.1.3. WebKitCSSMatrix mutable methods

The following method modifies the current matrix, returning the matrix it was invoked on.

When the setMatrixValue(transformList) method is invoked, the user agent must run the following steps:

  1. Return the result of the algorithm for the DOMMatrix setMatrixValue(transformList) method.

3.1.4. WebKitCSSMatrix immutable methods

The following methods do not modify the current matrix and return a new WebKitCSSMatrix object.

When the multiply(other) method is invoked, the user agent must run the following steps:

  1. Return the result of the algorithm for the DOMMatrixReadOnly multiply(other) method as a new WebKitCSSMatrix object.

When the inverse() method is invoked, the user agent must run the following steps:

  1. Return the result of the algorithm for the DOMMatrixReadOnly inverse() method as a new WebKitCSSMatrix object.

When the translate(tx, ty, tz) method is invoked, the user agent must run the following steps:

  1. Return the result of the algorithm for the DOMMatrixReadOnly translate(x, y, z) method as a new WebKitCSSMatrix object.

When the scale(scaleX, scaleY, scaleZ) method is invoked, the user agent must run the following steps:

  1. If scaleX or scaleZ are undefined, let scaleX or scaleZ be 1, respectively.

  2. If scaleY is undefined, let scaleY be the value of scaleX.

  3. Return the result of the algorithm for the DOMMatrixReadOnly scaleNonUniform(scaleX, scaleY, scaleZ) method as a new WebKitCSSMatrix object.

When the rotate(rotX, rotY, rotZ) method is invoked, the user agent must run the following steps:

  1. Let result be the resulting matrix initialized to the values of the current matrix.

  2. If rotY and rotZ are undefined, let rotZ be the value of rotX and let rotX and rotY be 0.

  3. Else, if any of rotX, rotY or rotZ are undefined, let the undefined argument be 0.

  4. Post multiply result by each of its 3 rotation matrices about the major axes, first X using rotX, then Y using rotY, then Z using rotZ.

  5. Return the result as a new WebKitCSSMatrix object.

The current matrix is not modified.

When the rotateAxisAngle(x, y, z, angle) method is invoked, the user agent must run the following steps:

  1. Return the result of the algorithm for the DOMMatrixReadOnly rotateAxisAngle(x, y, z, angle) method as a new WebKitCSSMatrix object.

When the skewX(sx) method is invoked, the user agent must run the following steps:

  1. Return the result of the algorithm for the DOMMatrixReadOnly skewX(sx) method as a new WebKitCSSMatrix object.

When the skewY(sy) method is invoked, the user agent must run the following steps:

  1. Return the result of DOMMatrixReadOnly skewY(sy) as a new WebKitCSSMatrix object.
